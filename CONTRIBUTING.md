# Contributing Guide（更新手順）

このリポジトリは、DS-ASTB2 のブラウザで GitHub Pages を開き、
`signage.mp4`（動画1本）を全画面でループ再生するための静的サイトです。

運用上の目的はシンプルで、**「動画を差し替えて、確実に反映させる」**ことです。

---

## 1. 基本方針（重要）
- **通常の変更は `signage.mp4` の差し替えのみ**にします。
- `index.html` / `manifest.webmanifest` / `sw.js` は表示や更新反映に関わるため、変更は最小限にし、必要なときだけ行います。
- 公開物です。個人情報・機微情報を含めないでください。

---

## 2. ファイル構成
- `index.html` : 表示ロジック（全画面・ループ・自動復旧・定時リロード）
- `signage.mp4` : 再生する動画（差し替え対象）
- `manifest.webmanifest` / `sw.js` / `icons/` : PWA（ホーム画面アイコン）関連

---

## 3. 動画更新の手順（いつもこれ）
### 3.1 更新フロー
1. 新しい動画を用意する（下の「動画の注意点」参照）
2. `signage.mp4` を **同名で置き換える**
3. commit → push
4. GitHub Pages の反映完了を待つ（Actions/Pagesで確認）
5. DS-ASTB2 は次のいずれかで反映されます  
   - すぐ反映：アプリ/ページを閉じて再起動（推奨）  
   - 放置で反映：定時リロード時刻に自動で再読み込み

---

## 4. 動画の注意点（トラブルを減らす）
### 4.1 回転（90°ずれる問題）
スマホ動画などは「回転メタデータ」が原因で、端末によって90°回転して見えることがあります。  
DS-ASTB2で回転した場合は、アップロード前にPC側で向きを焼き込みしてください（ffmpeg等）。

### 4.2 サイズ
- Webブラウザ経由のアップロードは失敗しやすいです（特に大きい動画）。
- うまくいかない場合は GitHub Desktop / git で push してください。

---

## 5. 変更してよい範囲（原則）
### 5.1 ふだん触る
- ✅ `signage.mp4`

### 5.2 触るときは注意
- ⚠️ `index.html`：再生/更新ロジックが壊れると表示できません  
- ⚠️ `sw.js`：動画をキャッシュする設定にすると更新が反映されなくなるため、原則いじらない  
- ⚠️ `manifest.webmanifest`：start_url/scopeを変えるとPWA起動が不安定になります

---

## 6. 動作確認（最低限）
- [ ] 公開ページをPC/スマホで開いて、動画が自動再生・ループする
- [ ] DS-ASTB2で再生できる（可能ならホーム画面アイコン起動でも確認）

---

## 7. 困ったとき（最短の復旧）
- 画面が真っ黒／動画が変わらない：
  1) DS-ASTB2側でページを閉じて開き直す（または再起動）  
  2) 直前にアップした `signage.mp4` が正しく置けているか確認  
  3) GitHub Pages の反映（Actions/Pages）が成功しているか確認
